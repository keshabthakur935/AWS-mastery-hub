Cloud Computing Glossary

AWS (Amazon Web Services)
A comprehensive, broadly adopted, and elastic cloud platform offered by Amazon.
EC2 (Elastic Compute Cloud)
A web service that provides resizable compute capacity in the cloud.
S3 (Simple Storage Service)
Object storage built to store and retrieve any amount of data from anywhere on the web.
VPC (Virtual Private Cloud)
A virtual network dedicated to your AWS account, logically isolated from other virtual networks in the AWS Cloud.
IAM (Identity and Access Management)
A web service that helps you securely control access to AWS resources.
Region
A physical location in the world where AWS has multiple Availability Zones.
Availability Zone (AZ)
One or more discrete data centers with redundant power, networking, and connectivity in an AWS Region.
Edge Location
Sites that Amazon CloudFront uses to cache copies of your content closer to your customers.
Shared Responsibility Model
Defines what AWS is responsible for (security *of* the cloud) and what the customer is responsible for (security *in* the cloud).
IaaS (Infrastructure as a Service)
Cloud computing model that provides virtualized computing resources over the internet.
PaaS (Platform as a Service)
Cloud computing model that provides a platform allowing customers to develop, run, and manage applications without the complexity of building and maintaining the infrastructure.
Partition Key
Part of a table's primary key, used to partition data across different storage nodes for scalability.
SaaS (Software as a Service)
Cloud computing model that provides software applications over the internet, on demand and typically on a subscription basis.
On-Premises
Software and technology that is installed and operates from a company's local server and computing infrastructure.
Hybrid Cloud
A cloud computing environment that uses a mix of on-premises, private cloud and public cloud services with orchestration between the platforms.
Public Cloud
Cloud services offered over the public internet and available to anyone who wants to purchase them.
Private Cloud
Cloud services offered either over the internet or a private internal network and only to select users instead of the general public.
Elasticity
The ability to acquire computing resources as you need them and release resources when you no longer need them.
Scalability
The ability of a system to handle a growing amount of work by adding resources to the system.
Fault Tolerance
The ability of a system to continue operating without interruption when one or more of its components fail.
Function
A script or program that runs in AWS Lambda.
High Availability
A characteristic of a system that aims to ensure an agreed level of operational performance, usually uptime, for a higher than normal period.
Load Balancer
Distributes incoming network traffic across a group of backend servers or resources.
Auto Scaling
Automatically adjusts the number of compute resources in your application in response to demand.
DNS (Domain Name System)
A hierarchical and decentralized naming system for computers, services, or other resources connected to the Internet or a private network.
CDN (Content Delivery Network)
A geographically distributed network of proxy servers and their data centers.
Database
An organized collection of structured information, or data, typically stored electronically in a computer system.
Dashboards
In CloudWatch, customizable home pages in the CloudWatch console that you can use to monitor your resources in a single view.
Serverless
A cloud-native development model that allows developers to build and run applications without having to manage servers.
Container
A standard unit of software that packages up code and all its dependencies so the application runs quickly and reliably from one computing environment to another.
DevOps
A set of practices that combines software development (Dev) and IT operations (Ops) to shorten the systems development life cycle and provide continuous delivery with high software quality.
Infrastructure as Code (IaC)
Managing and provisioning computer data centers through machine-readable definition files, rather than physical hardware configuration or interactive configuration tools.
Monitoring
The act of observing and checking the behavior or output of a system or process.
Logging
The process of recording events that occur in a computer system or network.
Metrics
Quantifiable measures used to track and assess the status of a specific business process.
Alerting
A system that notifies users or other systems when a specific event or condition occurs.
Alarms
In CloudWatch, an alarm watches a single metric over a time period you specify, and performs one or more actions based on the value of the metric relative to a threshold over a number of time periods.
Encryption
The process of converting information or data into a code, especially to prevent unauthorized access.
Compliance
Adhering to a set of rules, standards, or laws.
Security Group
Acts as a virtual firewall for your EC2 instances to control inbound and outbound traffic.
Network ACL (Access Control List)
An optional layer of security for your VPC that acts as a firewall for controlling traffic in and out of one or more subnets.
VPN (Virtual Private Network)
Extends a private network across a public network and enables users to send and receive data across shared or public networks as if their computing devices were directly connected to the private network.
Direct Connect
A cloud service solution that makes it easy to establish a dedicated network connection from your premises to AWS.
Route Table
Contains a set of rules, called routes, that are used to determine where network traffic is directed.
Rule
In AWS WAF, a condition that AWS WAF searches for in web requests. You can create your own rules or use managed rules.
Subnet
A logical subdivision of an IP network.
CIDR (Classless Inter-Domain Routing)
A method for allocating IP addresses and for IP routing.
NAT Gateway (Network Address Translation Gateway)
Enables instances in a private subnet to connect to the internet or other AWS services, but prevents the internet from initiating a connection with those instances.
Bastion Host
A server whose purpose is to provide access to a private network from an external network, such as the Internet.
Peering Connection
A networking connection between two VPCs that enables you to route traffic between them privately.
CloudFront
A fast content delivery network (CDN) service that securely delivers data, videos, applications, and APIs to customers globally with low latency, high transfer speeds, all within a developer-friendly environment.
Global Accelerator
A networking service that improves the availability and performance of your applications with local or global users.
CloudFormation
A service that helps you model and set up your Amazon Web Services resources so that you can spend less time managing those resources and more time focusing on your applications that run in AWS.
Elastic Beanstalk
An easy-to-use service for deploying and scaling web applications and services developed with Java, .NET, PHP, Node.js, Python, Ruby, Go, and Docker on familiar servers such as Apache, Nginx, Passenger, and IIS.
CodePipeline
A continuous delivery service that helps you automate your release pipelines for fast and reliable application and infrastructure updates.
CodeDeploy

A database that primarily relies on main memory for data storage, resulting in very fast response times (e.g., ElastiCache).
Instance Type
Defines the virtual hardware for an EC2 instance, including CPU, memory, storage, and networking capacity.
Internet Protocol (IP) Address
A numerical label assigned to each device connected to a computer network that uses the Internet Protocol for communication.
Key Management Service (KMS)
Makes it easy for you to create and manage cryptographic keys and control their use across a wide range of AWS services and in your applications.
Latency
The time delay between a cause and effect in a system, often referring to network delay.
Load Balancing
The process of distributing network traffic across multiple servers to improve application responsiveness and availability.
Log Group
A group of log streams that share the same retention, monitoring, and access control settings in CloudWatch Logs.
Log Stream
A sequence of log events from a single source in CloudWatch Logs.
Machine Learning (ML)
A type of artificial intelligence that allows software applications to become more accurate in predicting outcomes without being explicitly programmed.
Managed Policy
An IAM policy that is created and managed by AWS (AWS managed policy) or by you (customer managed policy).
Managed Rule Group
In AWS WAF, a pre-configured set of rules managed by AWS or AWS Marketplace sellers to protect against common threats.
Massively Parallel Processing (MPP)
A type of computing that uses many processors to perform a set of coordinated computations in parallel.
Metadata
Data that provides information about other data.
Metric
A variable that is measured and used to track the performance or health of a system.
Microservices Architecture
An architectural style that structures an application as a collection of loosely coupled, independently deployable services.
NAT Instance
An EC2 instance configured to perform Network Address Translation, allowing instances in a private subnet to access the internet.
Network Interface
A component that connects a computer to a computer network.
Object Key
The unique identifier for an object within an S3 bucket.
Origin Server
The source of content that a CDN (like CloudFront) retrieves and caches.
Parameter Store
A capability of AWS Systems Manager that provides secure, hierarchical storage for configuration data management and secrets management.
Persistent Storage
Storage that retains data even after the device is powered off (e.g., EBS, S3).
Pilot Light
A disaster recovery strategy where a minimal version of the environment is always running in the DR region, ready to be scaled up.
Pre-Signed URL
A URL that grants temporary access to a specific S3 object without requiring AWS credentials.
Private IP Address
An IP address used within a private network, not routable over the internet.
Public IP Address
An IP address that is routable over the internet.
Pub/Sub (Publish/Subscribe)
A messaging pattern where senders (publishers) send messages to a topic, and receivers (subscribers) receive messages from the topic.
Queue
A temporary storage location for messages, used to decouple senders and receivers in a distributed system.
Read Replica
A copy of a database instance that handles read traffic, offloading the primary instance and improving read performance.
Region
A physical location in the world where AWS has multiple Availability Zones.
Relational Database Service (RDS)
A managed service that makes it easy to set up, operate, and scale a relational database in the cloud.
Reserved Instance (RI)
An EC2 pricing model where you commit to a specific instance type and Region for a 1-year or 3-year term for significant discounts.
Root User
The account owner, created when you first set up your AWS account, with full administrative access.
Route Table
Contains a set of rules, called routes, that are used to determine where network traffic is directed within a VPC.
Routing Policy
A configuration in Route 53 that determines how DNS queries are responded to, enabling advanced traffic management.
Scalability
The ability of a system to handle a growing amount of work by adding resources to the system.
Schema
The organization or structure of a database.
Secrets Manager
Helps you protect access to your applications, services, and IT resources by enabling you to easily rotate, manage, and retrieve database credentials, API keys, and other secrets throughout their lifecycle.
Security Token Service (STS)
A web service that enables you to request temporary, limited-privilege credentials for IAM users or for users that you authenticate (federated users).
Server-Side Encryption (SSE)
Encrypting data at its destination by the service that receives it (e.g., S3, KMS, customer-provided keys).
Service Control Policies (SCPs)
A type of policy that you can use to manage permissions in your organization in AWS Organizations.
Service Discovery
The process of automatically detecting devices and services on a network.
Session Management
The process of securely handling multiple requests to a web-based application or service from a single user or client.
Simple Routing
A Route 53 routing policy that routes traffic to a single resource.
Site-to-Site VPN
A secure connection between your on-premises network and your Amazon VPC over the public internet.
Snapshot
A point-in-time backup of an EBS volume.
Static Content
Web content that does not change based on user interaction or other variables, typically HTML, CSS, JavaScript, images.
Static Website Hosting
Hosting a website with static content directly from an S3 bucket.
Storage Class
Defines the availability, durability, and cost of data stored in S3 (e.g., Standard, Standard-IA, Glacier).
Streaming Data
Data that is generated continuously by thousands of data sources, which typically send in the data records simultaneously, and in small sizes (kilobytes).
Subnet Mask
A 32-bit number that masks an IP address and divides the IP address into a network address and host address.
Target Group
A logical grouping of targets (e.g., EC2 instances) that an Elastic Load Balancer routes traffic to.
Throughput
The rate at which data is processed or transferred.
Time To Live (TTL)
A setting in a DNS record that tells a DNS resolver how long to cache a query before requesting a new update.
Topic
A logical access point that acts as a communication channel in Amazon SNS.
Traffic Management
The process of controlling and optimizing the flow of network traffic.
Transit Gateway
A network transit hub that you can use to interconnect your VPCs and on-premises networks.
Trigger
An AWS service or application that invokes a Lambda function.
User Data
A script or set of commands that you can provide when launching an EC2 instance, which runs automatically the first time the instance starts up.
Vendor Lock-in
A situation where a customer becomes dependent on a vendor for products and services, unable to move to another vendor without substantial costs or effort.
Versioning
A feature in S3 that allows you to keep multiple versions of an object in the same bucket.
Virtual Private Gateway (VPG)
The VPN concentrator on the Amazon side of a Site-to-Site VPN connection.
Warm Standby
A disaster recovery strategy where a fully functional, but scaled-down, version of the environment is always running in the DR region.
Weighted Routing
A Route 53 routing policy that routes traffic to multiple resources based on a weight you assign to each resource.
